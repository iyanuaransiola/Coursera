## Solution
1. Git is used to create local repository and can be used to independently worked on files while Github is for central or remote repository and allows for interactions of developers in various location on a particular projects.
2. Version Control System enables easy local access and repository storage in the local repository, it makes room for personal changes or modifications made to a file.
3. Since Git is used to keep local repositories it can use any language.
4. Gitlab, Github, Bitbucket, Stash, Codebase,Beanstalk.
5. A repository in Git basically mean a folder.
6. One can create a repository in Git by entering the command make directory "mkdir"
7. Git Commands
    a. ls means to list all contents in the directory
    b. pwd means to print working directory, and this will display what is presently worked on
    c. touch means to create file
    d. c: is to indicate a drive
    e. cd.. is to take to previous directory
    f. git status will show all that has been done in a directory
    g. init is the initialisation command 
    h. git log (commit log) is to show what has been successfully committed or captures at a certain time
    i. rm -m name of file   : this is to remove a particular file
    j. git push -u is to push upstream to remote repository
    k. git commit -m is for git to register or capture additional changes
8. Git is a version control system which possesses speed that helps in managing directories and tracking changes made in them, thereby coordinationg project workflow among team members.
9. .git directory is the folder that contains everything (database) about the project being worked on.
10. when .git Directory gets deleted the history is being captured, however the deleted directory cannot be retrieved from the local repository but it can be pulled from the remote server.
